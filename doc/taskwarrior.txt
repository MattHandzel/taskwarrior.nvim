==============================================================================
1. INTRODUCTION                                                *taskwarrior-intro*

taskwarrior.nvim turns the output of `task export` into a markdown buffer, lets you
edit it with any vim motion, and reconciles the diff back into Taskwarrior
on `:w`. Inspired by oil.nvim.

The on-disk representation is never persisted: each buffer is a virtual
scratch view of the live Taskwarrior database.

==============================================================================
2. REQUIREMENTS                                         *taskwarrior-requirements*

    - Neovim >= 0.9
    - Taskwarrior >= 2.6 (3.x supported)
    - Python >= 3.8 (optional; only needed for the `bin/taskmd` CLI and
      the live diff-preview virtual text. The default backend is pure Lua.)

==============================================================================
3. INSTALLATION                                        *taskwarrior-installation*

With lazy.nvim: >

    {
      "MattHandzel/taskwarrior.nvim",
      config = function() require("taskwarrior").setup({}) end,
    }
<
With packer.nvim: >

    use { "MattHandzel/taskwarrior.nvim",
          config = function() require("taskwarrior").setup({}) end }
<
                                                       *taskwarrior-rename*
Migrating from task.nvim ~

This plugin was previously published as `MattHandzel/task.nvim`. If you
were running v1.2.x or earlier, update your plugin spec to the new repo
name and replace `require("task")` with `require("taskwarrior")` in your
configuration. The `:Task*` commands and the `taskmd` filetype are
unchanged. Saved views, registered project mappings, and apply backups
are migrated automatically the first time the plugin loads.

4. QUICK START                                            *taskwarrior-quickstart*

    :Task                          open all pending tasks
    :Task project:career           filter
    :Task due.before:eow           tasks due this week

Edit any line. `:w` to sync. That is the whole loop.

5. COMMANDS                                                *taskwarrior-commands*

                                                                    *:Task*
:Task [filter]          Open task buffer with optional filter.

                                                              *:TaskFilter*
:TaskFilter [filter]    Change filter on the current buffer.

                                                                *:TaskSort*
:TaskSort <spec>        Change sort order. Examples: `due+`, `urgency-`,
                        `priority-`.

                                                               *:TaskGroup*
:TaskGroup [field]      Group by a field (`project`, `tag`) or `none` to
                        disable grouping.

                                                             *:TaskRefresh*
:TaskRefresh            Reload from Taskwarrior.

                                                                 *:TaskAdd*
:TaskAdd                Quick-capture a task in a floating window. Supports
                        inline fields: `Fix bug project:work priority:H +x`.

                                                                *:TaskUndo*
:TaskUndo               Reverse the last save. Best-effort — operations
                        Taskwarrior can't reverse (e.g. `task delete` on a
                        UUID) are reported as errors.

                                                                *:TaskHelp*
:TaskHelp               Show an in-buffer help reference.

                                                               *:TaskStart*
                                                                *:TaskStop*
:TaskStart / :TaskStop  Mark the task under the cursor as active/inactive.
                        Active tasks render with a `[>]` checkbox.

                                                                *:TaskSave*
                                                                *:TaskLoad*
:TaskSave <name>        Save the current filter+sort+group as a named view.
:TaskLoad [name]        Load a saved view (tab-completes).

                                                              *:TaskReview*
:TaskReview             Guided urgency walk. For each pending task: `k`
                        keep, `d` defer, `D` done, `m` modify, `g` go, `q`
                        quit.

                                                            *:TaskDelegate*
:TaskDelegate           Delegate task(s) to Claude. See |taskwarrior-delegate|.

                                                         *:TaskDiffPreview*
:TaskDiffPreview [on|off|toggle]
                        Toggle live virt-text diff annotations.

                                                            *:TaskBurndown*
                                                                *:TaskTree*
                                                             *:TaskSummary*
                                                            *:TaskCalendar*
                                                                *:TaskTags*
Visualisation commands. See |taskwarrior-views|.

                                                          *:TaskProjectAdd*
                                                       *:TaskProjectRemove*
                                                         *:TaskProjectList*
:TaskProjectAdd [name]   Register cwd as a project (see |taskwarrior-projects|).
:TaskProjectRemove       Unregister cwd.
:TaskProjectList         List all mappings.

                                                            *:TaskFeedback*
:TaskFeedback           Open a structured feedback buffer. Opt-in; needs
                        `feedback_endpoint` set in |taskwarrior-config|.

                                                              *:TaskAppend*
                                                             *:TaskPrepend*
:TaskAppend [text]      Append text to the description of the task under the
:TaskPrepend [text]     cursor. Without args, prompts for input. Buffer
                        keymaps `>>` and `<<`.

                                                           *:TaskDuplicate*
:TaskDuplicate          Duplicate the task under the cursor (fresh UUID,
                        status=pending, same project/tags/due). Buffer
                        keymap `yt`.

                                                               *:TaskPurge*
:TaskPurge [filter]     Irreversibly drop tasks from the database. Without
                        args, purges the task on the cursor (if it's
                        deleted) or prompts for a filter. Confirms before
                        acting. Buffer keymap `dD`.

                                                            *:TaskDenotate*
:TaskDenotate           Remove an annotation from the task under the cursor.
                        If the task has more than one annotation, prompts
                        for which to remove. Buffer keymap `gA`.

                                                         *:TaskModifyField*
:TaskModifyField <name> Pick a new value for a single field of the task under
                        the cursor. Known fields: `project`, `priority`,
                        `due`, `tag`. Falls back to free-form input for
                        anything else. Also accessible via buffer keymaps
                        `MM` (project), `Mp` (priority), `MD` (due),
                        `Mt` (tag).

                                                         *:TaskBulkModify*
:'<,'>TaskBulkModify <spec>
                        Apply the same modify spec to every task in the
                        selected range. Each task is modified in its own
                        subprocess so Taskwarrior's bulk-confirmation
                        prompt is sidestepped. Example: >
                            :'<,'>TaskBulkModify +triage
<
                                                              *:TaskReport*
:TaskReport [name]      Open a named Taskwarrior report. Built-in reports:
                        `next`, `active`, `overdue`, `recurring`, `waiting`,
                        `unblocked`, `ready`, `blocked`, `completed`,
                        `today`, `week`, `noproject`. Without args, opens a
                        selection prompt.

                                                               *:TaskGraph*
:TaskGraph              Render the `depends:` graph of pending tasks as a
                        Mermaid flowchart in a markdown buffer.

                                                               *:TaskInbox*
:TaskInbox              Walk tasks added in the last 24h with no project, no
                        due date, and no tags. For each: set project,
                        schedule, tag, defer, drop, skip, or quit.

                                                              *:TaskExport*
:TaskExport [path]      Write the current task buffer out as plain markdown
                        (UUIDs stripped, header comment removed). Useful for
                        archival or sharing.

                                                                *:TaskSync*
:TaskSync               Run `task sync` asynchronously with progress and
                        error reporting. Offers a retry on failure.

                                                               *:TaskFloat*
:TaskFloat [filter]     Open the task buffer in a centered floating window
                        instead of a split. `q` to dismiss.

                                                       *:TaskLinkChildren*
                                                     *:TaskUnlinkChildren*
:TaskLinkChildren       Treat tasks indented directly below the cursor task
                        as `depends:` of it. Inverse: :TaskUnlinkChildren
                        drops those same dependencies.

                                                       *:TaskQueryRefresh*
:TaskQueryRefresh       Re-render every `<!-- taskmd query: ... -->` block
                        in the current buffer. Blocks also refresh
                        automatically on BufReadPost / BufWritePost of
                        `*.md` / `*.markdown` buffers.

6. KEYBINDINGS                                          *taskwarrior-keybindings*

Global (configurable; see |taskwarrior-config|): ~
    <leader>tt          Open task buffer.
    <leader>ta          Quick-capture.
    <leader>tpa         Register cwd as a project.

Buffer-local (inside a task buffer): ~
    <CR>                Cycle state: pending -> active -> done -> pending.
    o / O               New task line below / above.
    dd                  Mark done (or delete, per |taskwarrior-config|
                        `on_delete`).
    yy + p              Duplicate a task (native vim; see also `yt`).
    ga                  Annotate the task under the cursor.
    gA                  Remove an annotation (|:TaskDenotate|).
    gm                  Modify fields via free-form input.
    gf                  Show `task info` in a bordered float (`q` / <Esc>
                        to dismiss).
    MM / MP             Pick a project from existing ones (|:TaskModifyField|).
    Mp                  Pick priority: H / M / L / (clear).
    MD                  Pick due date: today / tomorrow / eow / custom…
    Mt                  Pick tag from existing ones (or add a custom).
    >>                  Append to description (|:TaskAppend|).
    <<                  Prepend to description (|:TaskPrepend|).
    yt                  Duplicate task (|:TaskDuplicate|).
    dD                  Purge task (irreversible, confirms).
    <leader>tf          Change filter interactively.
    <leader>ts          Change sort interactively.
    <leader>tg          Change grouping interactively.

==============================================================================
7. SYNTAX                                                    *taskwarrior-syntax*

Tasks are rendered as GFM checkboxes with Taskwarrior-native metadata: >

    - [ ] Fix login bug project:Work priority:H due:2026-04-01 +urgent
    - [x] Completed task                    <!-- uuid:ab05fb51 -->
    - [>] Active task

Fields: `project:`, `priority:` (H/M/L), `due:`, `scheduled:`, `recur:`,
`wait:`, `until:`, `effort:`, `depends:`, plus any UDA.

Tags: `+tagname`. Hyphens are allowed (`+my-tag`).

UUID comments are concealed (conceallevel=3) so they never interfere with
editing. They travel with the line through any vim operation that preserves
that line.

8. CONFIGURATION                                             *taskwarrior-config*

Full schema with defaults: >

    require("taskwarrior").setup({
      on_delete         = "done",     -- or "delete"
      confirm           = true,
      sort              = "urgency-",
      group             = nil,
      fields            = nil,
      backend           = "lua",      -- or "python"
      capture_key       = "<leader>ta",
      open_key          = "<leader>tt",
      filter_key        = "<leader>tf",
      sort_key          = "<leader>ts",
      group_key         = "<leader>tg",
      project_add_key   = "<leader>tpa",
      filters           = {},
      projects          = {},
      icons             = true,
      border_style      = "rounded",
      capture_width     = nil,
      capture_height    = 3,
      capture_confirm_close = true,  -- prompt before discarding text on <Esc>
      group_separator   = true,
      animation         = true,
      clamp_cursor      = true,
      day_start_hour    = 4,
      auto_backup       = true,       -- see |taskwarrior-safety|
      urgency_coefficients = {},
      urgency_value_mappers = nil,
      custom_urgency    = nil,
      feedback_endpoint = false,
      delegate = {
        command            = "claude",
        flags              = "",      -- see |taskwarrior-delegate|
        system_prompt_file = nil,
        model              = nil,
        height             = 0.5,
      },
      -- Per-tag highlight overrides (see |taskwarrior-tag-colors|).
      tag_colors = {},                -- e.g. { ["+urgent"] = "ErrorMsg" }
      -- Urgency breakpoints for virtual text + views (|taskwarrior-urgency-colors|).
      urgency_colors = {
        { threshold = 8, hl = "TaskPriorityH" },
        { threshold = 4, hl = "TaskPriorityM" },
        { threshold = 0, hl = "TaskPriorityL" },
      },
      -- Category gates for vim.notify (|taskwarrior-notifications|).
      notifications = {
        start = true, stop = true, modify = true, apply = true,
        review = true, capture = true, delegate = true, view = true,
        error = true, warn = true,
      },
      -- Auto-stop running task timers when nvim is idle (|taskwarrior-granulation|).
      granulation = {
        enabled = false,
        idle_ms = 5 * 60 * 1000,
        notify_on_stop = true,
      },
      -- Slot functions for the task-buffer header stats line.
      header_stats = nil,             -- or { fn, fn, ... }
      -- Nerd-font icons: true|nil (auto), false (force ASCII), or
      -- partial override table. See |taskwarrior-icons|.
      icons = true,
      -- Render "today" / "tomorrow" / "in 3d · Fri" as virt-text.
      relative_dates = true,
      -- Show the numeric urgency score as a right-side chip.
      show_urgency = false,
      -- If show_urgency is true, prefix with an 8-band unicode bar.
      urgency_bar = true,
      -- Render an OVERDUE pill for tasks past their due date.
      overdue_badge = false,
    })
<

See |taskwarrior-safety| for auto_backup and |taskwarrior-delegate| for delegate.flags.

                                                        *taskwarrior-tag-colors*
Per-tag colors ~

Override the highlight group used for a specific tag: >
    require("taskwarrior").setup({
      tag_colors = {
        ["+urgent"]  = "ErrorMsg",
        ["+someday"] = "Comment",
        ["+waiting"] = { fg = "#9399b2", italic = true },  -- inline hl spec
      },
    })
<
Values may be either a highlight-group name (string) or a table spec passed
to |nvim_set_hl()|. Unmatched tags fall through to the default `TaskTag`
highlight.

                                                   *taskwarrior-urgency-colors*
Urgency color breakpoints ~

The virtual-text urgency number and the view renderers pick a highlight by
matching the urgency against a list of breakpoints (first match wins): >
    require("taskwarrior").setup({
      urgency_colors = {
        { threshold = 12, hl = "ErrorMsg" },
        { threshold = 6,  hl = "WarningMsg" },
        { threshold = 0,  hl = "Comment" },
      },
    })
<

                                                   *taskwarrior-notifications*
Silencing notifications ~

Turn off specific categories with:  >
    require("taskwarrior").setup({
      notifications = { apply = false, modify = false },
    })
<
Recognized categories: `start`, `stop`, `modify`, `apply`, `review`,
`capture`, `delegate`, `view`, `error`, `warn`. Setting an unknown key is
a setup() error.

                                                          *taskwarrior-icons*
Icons (Nerd Font) ~

taskwarrior.nvim renders task lines with nerd-font glyphs by default
(`icons = true`). Set `icons = false` to force ASCII (accessibility
path) or `icons = "auto"` to detect via `vim.g.have_nerd_font`. The
default-on behavior matches the option's wording and avoids the
common gotcha where users with an NF terminal forget to set the
global flag and see no icons. Override slots individually: >
    require("taskwarrior").setup({
      icons = {
        priority_h   = "!!!",    -- any slot in lua/taskwarrior/icons.lua
        status_started = ">>",
      },
    })
<
Or force the full ASCII set (accessibility path): >
    require("taskwarrior").setup({ icons = false })
<
Slots (see `lua/taskwarrior/icons.lua`): `checkbox_pending`,
`checkbox_started`, `checkbox_done`, `checkbox_blocked`, `priority_h`,
`priority_m`, `priority_l`, `status_started`, `status_overdue`,
`status_note`, `project`, `tag`, `due_normal`, `due_today`,
`due_overdue`, `recur`, `effort`, `urg_1`..`urg_8`, `badge_overdue`.

Checkbox slots are painted as a 5-cell `virt_text` overlay over the
literal `- [ ]` / `- [>]` / `- [x]` in the source. The markdown text
itself is never rewritten — the Taskwarrior round-trip parser depends
on those literals. The overlay is automatically suppressed when no
nerd font is available AND no slot override is configured (the ASCII
fallback would just redraw the source).

                                                 *taskwarrior-relative-dates*
Contextual due-date labels ~

Each `due:YYYY-MM-DD` field in a task line gets a right-aligned
virt-text label (`today`, `tomorrow`, `in 3d · Fri`, `next Mon`,
`2d overdue`, ...). The literal date in the buffer text is never
rewritten — `:w` continues to round-trip correctly.

Disable: `relative_dates = false`.

                                                      *taskwarrior-urgency-bar*
Urgency display ~

**Off by default.** Most users find the relative-date label and
sign-column priority glyphs more actionable than a raw urgency
score. Enable with: >
    require("taskwarrior").setup({ show_urgency = true })
<
When enabled, each task's urgency is rendered as an 8-band unicode
block (`▁▂▃▄▅▆▇█`) followed by the numeric score. Suppress the bar
and keep the bare number with `urgency_bar = false`.

                                                    *taskwarrior-overdue-badge*
OVERDUE pill ~

**Off by default.** The relative-date label already says "Nd overdue"
in the same slot, so the pill is mostly redundant. Enable for a
high-contrast alarm effect: >
    require("taskwarrior").setup({ overdue_badge = true })
<

                                                    *taskwarrior-granulation*
Auto-stop on idle ~

When a Taskwarrior task is `start`ed (active), users often forget to stop
it. Enable the idle watcher: >
    require("taskwarrior").setup({
      granulation = { enabled = true, idle_ms = 300000 },  -- 5 min
    })
<
After `idle_ms` with no cursor/focus activity, every active task is
`task <uuid> stop`ped and any open task buffers refresh. A notification
fires when tasks are auto-stopped unless `notify_on_stop = false`.

                                                       *taskwarrior-reports*
Named reports ~

`:TaskReport <name>` opens the Taskwarrior report named `<name>`:

    next         status:pending, sorted by urgency
    active       currently started (+ACTIVE)
    overdue      pending + due date in the past
    ready        pending, not blocked, not waiting
    unblocked    pending, not blocked
    waiting      scheduled for later (+WAITING)
    recurring    recurring masters
    blocked      pending + blocked
    completed    already done, newest first
    today        due before tomorrow
    week         due before end-of-week
    noproject    pending tasks with no project set

                                                  *taskwarrior-query-blocks*
Embedded query blocks in markdown ~

Any markdown buffer can host a live Taskwarrior view via an HTML-comment
block: >
    <!-- taskmd query: status:pending +urgent | sort:due+ | group:project -->
    (content auto-filled)
    <!-- taskmd endquery -->
<
Blocks refresh on BufReadPost / BufWritePost, and on demand via
`:TaskQueryRefresh`. The block body is read-only — to edit a task, open
`:Task <filter>` (or use the Telescope picker). Multiple blocks per buffer
are supported.

11. PROJECT AUTO-FILTER                                   *taskwarrior-projects*

`:TaskProjectAdd career` (run inside `~/projects/career-dev/`) maps the
current directory to `project:career`. Subsequent `:Task` with no filter,
when invoked from inside that directory, auto-applies `project:career`.

Persists to `stdpath("data")/taskwarrior_nvim_projects.json`.

12. DELEGATION                                             *taskwarrior-delegate*

`:TaskDelegate` opens a popup (context, flags, model, system-prompt-file)
and runs the configured `delegate.command` in a visible bottom split.

Visual-range support: select multiple task lines and run `:TaskDelegate` to
delegate them all in one session.

Sub-commands: ~
    copy            Copy the assembled prompt to `+` register.
    copy-command    Copy the full shell invocation to `+` register.

                                                    *taskwarrior-delegate-flags*
The default `delegate.flags` is the empty string. For power users who want
unattended invocations, set it explicitly in setup(): >

    delegate = { flags = "--dangerously-skip-permissions" },
<
This flag is not the default because it skips tool-permission checks in
Claude Code and shouldn't be invisible.

14. DATA SAFETY                                             *taskwarrior-safety*

`:w` issues real `task modify`, `task done`, `task add`, `task delete`
commands. There is no staging; the buffer is the edit.

                                                      *taskwarrior-auto-backup*
If `auto_backup = true` (default), taskwarrior.nvim copies your Taskwarrior data
directory to >

    stdpath("data")/taskwarrior.nvim/backups/<YYYY-MM-DD-HHMMSS>/
<
immediately before applying any save. Backups are kept rolling — the ten
newest are retained, older ones are pruned. Disable by setting
`auto_backup = false`.

taskwarrior.nvim also refuses to apply a markdown file whose header line has been
deleted or corrupted, unless the `taskmd` CLI is invoked with `--force`.
This prevents the "silent delete every pending task" failure mode.

You should additionally keep your Taskwarrior database under version
control or a regular backup — taskwarrior.nvim's backups are convenience, not a
substitute for real backups.

15. CLI                                                        *taskwarrior-cli*

The bundled `bin/taskmd` script is usable standalone: >

    taskmd render project:Inbox --sort=due+
    taskmd render --group=project
    taskmd apply tasks.md
    taskmd apply tasks.md --dry-run
    taskmd completions
<
Useful for piping rendered tasks through grep/fzf, or integrating with
editors other than Neovim.

16. TROUBLESHOOTING                                 *taskwarrior-troubleshooting*

Error "Not allowed to change text or change window" when submitting
:TaskAdd with nvim-cmp installed ~

    Fixed in the current version. If it recurs, reload the plugin
    (`:Lazy reload taskwarrior.nvim`) and file a bug with the stack trace.

Error "Invalid buffer id" after applying changes ~

    Caused by a stale per-buffer autocmd firing after the buffer was wiped.
    Fixed in the current version.

Swap-file warnings when reopening a task buffer ~

    Task buffers should set `swapfile=false` automatically. If you still
    see warnings, confirm you are on the current version and file a bug.

Completion popup doesn't appear ~

    :TaskAdd uses omnifunc. Press <C-x><C-o> manually; <Tab> is
    pre-wired to trigger it. If nothing appears, run `:checkhealth taskwarrior`
    to confirm Taskwarrior is reachable.

                                                     *taskwarrior-statusline*
STATUSLINE / LUALINE                                                       ~

taskwarrior.nvim ships a ready-to-use statusline component at
`lua/taskwarrior/statusline.lua`. It renders the active task, overdue count,
and next upcoming due date, cached for 15s so it never stalls your prompt.

Lualine: >
    require("lualine").setup({
      sections = {
        lualine_x = {
          require("taskwarrior.statusline").component,
          "filetype",
        },
      },
    })
<
Heirline: >
    local tw = require("taskwarrior.statusline")
    {
      provider = function() return tw.render() end,
      update = { "User", pattern = "TaskwarriorRefresh" },
    }
<
Raw statusline: >
    vim.o.statusline = '%!v:lua.require("taskwarrior.statusline").render()'
<
Invalidate the cache explicitly (e.g. after external task CLI edits): >
    require("taskwarrior.statusline").invalidate()
<

                                                     *taskwarrior-dashboard*
ALPHA / DASHBOARD WIDGETS                                                  ~

`lua/taskwarrior/dashboard.lua` exposes `top_urgent(n)` — returns a list of
pretty-printed task lines for use in a startup screen. Safe to call at
UIEnter: if Taskwarrior isn't installed or the database is empty it returns
a single informational line.

alpha.nvim: >
    local dashboard = require("alpha.themes.dashboard")
    dashboard.section.footer.val =
      require("taskwarrior.dashboard").top_urgent(5)
<
dashboard.nvim: >
    require("dashboard").setup({
      custom_section = {
        tasks = {
          description = require("taskwarrior.dashboard").top_urgent(5),
          command = ":Task",
        },
      },
    })
<
